# モジュールマッピング（現コード → 目標モジュール）

最終更新: 2026-06-02

関連: 戦略は [リファクタリング方針.md](リファクタリング方針.md)、目標設計は [目標アーキテクチャ図.md](目標アーキテクチャ図.md)。本資料は両者をつなぐ「実作業の地図」。

## 目的

現コードの各メソッドが、目標モジュール（core / data / platform / feature / ui / app）のどこへ行くかを**漏れなく**対応づける。リライト時に処理を取りこぼさないための地図。

## 1. メソッド通し番号 方針

スクリプトでメソッド一覧を抽出し、**CSV側でIDを採番する**。ソースコードへのコメント埋め込みは行わない。

- **通し番号**: 全ファイル横断の単一連番（001, 002, …）。CSV の `通しID` 列に記載。
- **対象**: private 含む全 `fun` ＋ `@Composable` 関数すべて（取りこぼし防止のため網羅）。ただし**匿名オブジェクト/ラムダ内で所属と一体の関数（独立してモジュール移動しないもの。例: BroadcastReceiver の `onReceive`）は対象外**。
- **抽出順**: MainActivity → MainViewModel → data/（ShortcutModel → ShortcutRepository → MemoRepository → SettingsRepository → PremiumManager → BackupManager → CalendarRepository）→ ui/screens/各画面 → ルート直下のその他。

## 2. マッピング表の形式

向きは **A（現在地起点）を主**とし、最後に **B（目標起点）の超要約**を付ける。

### A. 現在地起点（本体・CSV出力）

データは CSV で出力し、手で Excel 化する。CSVヘッダ:

```
通しID,ファイル,メソッド,現責務,行き先モジュール,純粋度,状態,備考
```

- **純粋度**: ◎=純粋 / △=Android依存だが共通化候補 / ✕=配管（書き直さず移植）
- **状態**: 未着手 / 完了 / 削除
- 移植時に分割・名称変更等が生じた場合はその時点で備考に追記する
- 出力先: `doc/モジュールマッピング.csv`（WPS保存後の SJIS→UTF8 変換は `E:\programing\bat\csv_utf8.bat` 参照）

### B. 目標起点（要約・マッピング完了後に記入）

各モジュールに「主に何が入るか」を1〜2行で。

| モジュール | 主な中身（現コードのどこ由来か） |
|---|---|
| core/layout | （後で記入） |
| core/shortcut | |
| core/premium | |
| core/navigation | |
| data | |
| platform | |
| feature/launcher（home, shortcutselect） | |
| feature/screens（settings, calendar, memo, allapps） | |
| ui | |
| app | |

## 3. 進め方

スクリプトでメソッド一覧を一括抽出 → CSVに採番 → 行き先モジュール・純粋度を記入していく。
