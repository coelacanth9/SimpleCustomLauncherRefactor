## 発見された設計漏れ（設計書更新要）

### 漏れ1: ShortcutSelectViewModel が Phase7 設計書に未記載

ShortcutSelectViewModel に存在するビジネスロジック:

**HomeViewModel と重複するロジック（UseCase に統合して重複解消が必要）:**

| ShortcutSelectViewModel | HomeViewModel | UseCase |
|---|---|---|
| deleteRow() | deleteRow() | RowUseCase.deleteRow() |
| changeColumns() | changeRowColumns() | RowUseCase.changeRowColumns() |
| changeTextOnly() | changeRowTextOnly() | RowUseCase.changeTextOnly()（追加） |
| clearSlot() | clearSlot() | EditSlotUseCase.clearSlot()（新規） |
| changeColors() | changeSlotColors() | EditSlotUseCase.changeColors()（新規） |

**ShortcutSelectViewModel 固有のロジック（PlaceShortcutUseCase として新規作成が必要）:**
- placeApp(packageName, label) — 重複チェック + saveShortcut + savePlacement
- placeInternalFeature(type, label) — 重複チェック + saveShortcut + savePlacement
- placeDeviceSettings(label, settingsAction) — 重複チェック + saveShortcut + savePlacement
- placeContact(name, phoneNumber, type, targetPackage) — 重複チェック + saveShortcut + savePlacement
- placeIntent(shortLabel, packageName, shortcutId) — 重複チェック + saveShortcut + savePinShortcutInfo + savePlacement
- swapShortcuts(targetShortcut) — 入れ替えロジック
- updateRowFixedHeight()（private）— DATE_DISPLAY→56dp / TIME_DISPLAY→80dp のビジネスルール
- placeItem()（private）— updateRowFixedHeight + savePlacement のまとめ

### 漏れ2: SettingsViewModel.resetToDefault() が設計書に未記載

`resetToDefault()` が `shortcutRepository.resetToDefault()` を直接呼んでいる。
Step3 で ShortcutRepository.resetToDefault() は UseCase に移動されるため、
SettingsViewModel も ApplyDefaultLayoutUseCase を呼ぶよう変更が必要。

### 漏れ3: BackupManager.restoreFromJson() の update{} 欠如

`restoreFromJson()` 内の書き込みシーケンス:
```
clearAllLayout()
shortcuts.forEach { saveShortcut(it) }  // N回 StateFlow 更新
placements.forEach { savePlacement(it) }  // N回 StateFlow 更新
saveLayoutConfig(...)  // 1回 StateFlow 更新
```
`update{}` なしで呼んでいるため、復元中に不完全な状態が StateFlow に見える。
`repository.update {}` でまとめる必要がある。

### 漏れ4: MainActivity.applyDefaultLayout() 直接呼び出し

行 128: `shortcutRepository.applyDefaultLayout()` を直接呼んでいる。
Step6 で UseCase 経由に変更する。
